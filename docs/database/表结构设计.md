# AI 网关表结构设计

## 1. 设计原则
- 统一审计字段：`created_at`、`updated_at`、`created_by`、`updated_by`。
- 统一软删除：`is_deleted`。
- 配置类数据采用 JSON 字段，减少早期频繁改表成本。
- 高写入日志与调用明细分区或按时间归档。

## 2. 核心实体分组
- 系统管理：用户、角色、权限、部门、操作日志、系统配置。
- 供应商管理：供应商、供应商凭证、供应商监控快照、供应商折扣策略。
- 模型管理：模型、模型版本、模型测试记录。
- 应用管理：应用、应用密钥、应用授权关系。
- 路由规则：路由规则、路由命中日志。
- 计费成本：计费规则、调用明细、账单汇总、对账单。
- 安全监控：安全策略、告警事件、缓存记录。
- 可选扩展：插件、灰度计划、通知模板与发送记录。

## 3. 关键关系
- 一个供应商可关联多个模型。
- 一个应用可授权多个模型（多对多）。
- 路由规则可指向供应商与模型。
- 调用明细关联应用、模型、供应商，并可关联计费规则。
- 告警可关联应用/模型/供应商等业务对象（`biz_type + biz_id`）。
- 供应商折扣策略可按供应商/模型/应用作用域生效，调用时生成折扣快照。

## 4. 索引建议
- 高频查询组合索引：
  - 调用明细：`(app_id, request_time)`、`(provider_id, request_time)`。
  - 路由命中：`(rule_id, hit_time)`。
  - 告警事件：`(level, status, created_at)`。
  - 操作日志：`(user_id, created_at)`、`(trace_id)`。
  - 折扣策略：`(provider_id, effective_from, effective_to)`。
- 唯一索引：
  - `users.username`
  - `apps.app_id`
  - `providers.code`
  - `models.model_code`

## 5. 分阶段落库建议
- 第一期：`providers`、`models`、`apps`、`routing_rules`、`gateway_requests`。
- 第二期：`billing_rules`、`billing_statements`、`alerts`、`security_policies`、`provider_discount_policies`。
- 第三期：插件、灰度、通知等增强模块。
