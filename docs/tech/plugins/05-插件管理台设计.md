# 插件管理台设计

## 1. 管理目标

管理台不应只做“插件名称 + 状态”。

它需要覆盖：

1. 插件包管理
2. 插件实例管理
3. 插件绑定管理
4. 插件配置版本管理
5. 插件执行观测

## 2. 页面结构建议

### 2.1 插件包

字段：

- 插件 ID
- 名称
- 版本
- 来源
- 入口类
- 能力列表
- 兼容版本
- 校验状态
- 当前状态

操作：

- 上传插件包
- 查看 manifest
- 禁用包
- 标记废弃

### 2.2 插件实例

字段：

- 实例编码
- 实例名称
- 所属插件包
- 状态
- 失败策略
- 超时
- 并发限制
- 当前生效配置版本

操作：

- 新建实例
- 编辑配置
- 发布新版本
- 暂停/启用

### 2.3 插件绑定

字段：

- 实例
- 作用域类型
- 作用域对象
- 执行阶段
- 顺序
- 灰度策略
- 状态

操作：

- 新增绑定
- 调整顺序
- 启用/禁用

### 2.4 观测面板

指标：

- 调用次数
- 成功率
- 平均耗时
- 超时次数
- 熔断状态
- 最近错误

### 2.5 调试页

建议支持：

- 输入上下文样本
- 选择实例
- 模拟执行
- 输出执行结果

## 3. 权限模型建议

至少拆成三类权限：

- 查看插件
- 管理插件配置
- 发布/启停插件

发布和启停应属于高权限动作。

## 4. 管理台交互原则

1. 管理插件实例，而不是直接编辑 jar 包
2. 配置变更必须版本化
3. 绑定关系必须可视化
4. 高风险插件启停必须可审计
