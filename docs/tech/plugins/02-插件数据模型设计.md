# 插件数据模型设计

## 1. 设计原则

插件模型必须区分：

1. 插件包是什么
2. 插件实例在哪里启用
3. 插件绑定到了哪里
4. 插件运行结果如何

不建议把这些概念全部塞进一张 `plugins` 表。

## 2. 推荐表设计

### 2.1 plugin_packages

存插件包定义。

| 字段 | 说明 |
|---|---|
| id | 主键 |
| plugin_id | 稳定插件标识，如 `content-filter` |
| plugin_name | 展示名称 |
| version | 版本号 |
| source_type | `BUILTIN` / `LOCAL_JAR` / `REMOTE_REGISTRY` |
| artifact_path | 插件包路径 |
| manifest_json | manifest 原文 |
| entrypoint | 入口类 |
| checksum | 校验值 |
| signature | 签名信息 |
| compatibility_json | 兼容矩阵 |
| status | `ACTIVE` / `DISABLED` / `DEPRECATED` |

### 2.2 plugin_capabilities

存插件声明的能力。

| 字段 | 说明 |
|---|---|
| id | 主键 |
| package_id | 所属插件包 |
| capability_code | 能力编码，如 `gateway-filter` |
| extension_point | 扩展点编码 |
| order_hint | 默认顺序 |
| config_schema_json | 配置 schema |

### 2.3 plugin_instances

存插件实例。

| 字段 | 说明 |
|---|---|
| id | 主键 |
| package_id | 所属插件包 |
| instance_code | 实例编码 |
| instance_name | 实例名称 |
| config_json | 实例配置 |
| status | `DRAFT` / `ACTIVE` / `PAUSED` / `ARCHIVED` |
| fail_mode | `FAIL_OPEN` / `FAIL_CLOSE` |
| timeout_ms | 超时 |
| concurrency_limit | 并发限制 |

### 2.4 plugin_bindings

存实例绑定关系。

| 字段 | 说明 |
|---|---|
| id | 主键 |
| instance_id | 插件实例 |
| scope_type | `GLOBAL` / `APP` / `RULE` / `PROVIDER` / `MODEL` / `POOL` |
| scope_ref_id | 作用域对象 ID |
| stage_code | 执行阶段 |
| order_no | 顺序 |
| match_expression | 可选匹配表达式 |
| gray_config_json | 灰度策略 |
| status | `ACTIVE` / `DISABLED` |

### 2.5 plugin_execution_logs

存插件执行记录。

| 字段 | 说明 |
|---|---|
| id | 主键 |
| trace_id | 请求追踪 ID |
| instance_id | 插件实例 |
| stage_code | 执行阶段 |
| scope_type | 作用域类型 |
| scope_ref_id | 作用域对象 |
| result_status | `SUCCESS` / `SKIPPED` / `FAILED` / `TIMEOUT` / `OPEN_CIRCUIT` |
| duration_ms | 执行耗时 |
| error_code | 错误码 |
| error_message | 错误信息 |
| snapshot_json | 输入输出摘要 |
| created_at | 创建时间 |

### 2.6 plugin_config_versions

存插件配置变更历史。

| 字段 | 说明 |
|---|---|
| id | 主键 |
| instance_id | 插件实例 |
| version_no | 版本号 |
| version_code | 配置摘要 |
| config_json | 配置快照 |
| operator_id | 操作人 |
| created_at | 创建时间 |

## 3. Manifest 建议结构

```json
{
  "pluginId": "content-filter",
  "name": "Content Filter",
  "version": "1.0.0",
  "entrypoint": "org.chobit.knot.plugin.content.ContentFilterPlugin",
  "capabilities": [
    {
      "code": "gateway-filter",
      "extensionPoint": "gateway.pre-route",
      "order": 100
    }
  ],
  "configSchema": {
    "type": "object",
    "properties": {
      "mode": { "type": "string" }
    }
  }
}
```

## 4. 兼容性建议

每个插件包都应声明：

- 最低内核版本
- 最高已验证内核版本
- 需要的 SPI 版本

否则未来内核升级时无法做兼容校验。
