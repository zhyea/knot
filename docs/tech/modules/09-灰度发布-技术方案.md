# 灰度发布模块技术方案

## 1. 架构设计
- 子模块：灰度计划、流量分配、健康评估、自动回滚。
- 组件：`GrayPlanner`、`TrafficSplitter`、`RollbackManager`。

## 2. 数据设计
- 核心表：`gray_plans`。
- 推荐新增：`gray_plan_steps`、`gray_plan_events`。

## 3. 接口设计
- `POST /api/release/gray-plans`
- `POST /api/release/gray-plans/{id}/publish`
- `POST /api/release/gray-plans/{id}/rollback`

## 4. 关键流程
- 草稿 -> 审批 -> 分阶段放量（10/30/50/100）-> 监控评估 -> 完成或回滚。

## 5. 异常与性能
- 当错误率超过阈值自动降回上一阶段。
