# 应用管理模块技术方案

## 1. 架构设计
- 子模块：应用档案、认证授权、额度频控、调用统计。
- 组件：
  - `AppAuthService`：AppKey/AppSecret 验签。
  - `AppQuotaService`：额度扣减与预警。
  - `AppPermissionService`：应用可调用模型列表管理。

## 2. 数据设计
- 核心表：`apps`、`app_credentials`、`app_model_permissions`。
- 推荐新增：`app_quota_usage_daily`（按天累计）。

## 3. 接口设计
- `GET/POST/PUT /api/apps`
- `PUT /api/apps/{id}/quota`
- `GET /api/apps/{id}/metrics`

## 4. 关键流程
- 应用调用：验签 -> IP 白名单检查 -> 权限检查 -> 频控检查 -> 路由。
- 密钥重置：生成新密钥 -> 旧密钥宽限期并存 -> 到期失效。

## 5. 异常与性能
- 高频验签结果短 TTL 缓存。
- 错误码区分：鉴权失败、权限拒绝、额度不足、频控触发。
