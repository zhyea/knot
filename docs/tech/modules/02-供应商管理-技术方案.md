# 供应商管理模块技术方案

## 1. 架构设计
- 子模块：供应商档案、凭证中心、配额频控、监控、折扣策略。
- 组件：
  - `ProviderClientFactory`：按供应商类型构建调用客户端。
  - `CredentialEncryptor`：KMS/本地密钥加密。
  - `DiscountEngine`：折扣规则命中与金额计算。

## 2. 数据设计
- 核心表：`providers`、`provider_credentials`、`provider_discount_policies`、`provider_discount_snapshots`。
- 推荐新增：`provider_metrics_minute`（分钟级聚合）。

## 3. 接口设计
- `GET/POST/PUT /api/providers`
- `PUT /api/providers/{id}/credential`
- `GET/POST/PUT /api/providers/{id}/discount-policies`

## 4. 关键流程
- 凭证轮换：提交新凭证 -> 健康检查 -> 原子切换 -> 审计记录。
- 折扣计算：请求路由命中后 -> 查可用折扣策略 -> 计算折扣 -> 写入 `provider_discount_snapshots`。

## 5. 异常与性能
- 供应商超时熔断 + 指数退避重试。
- 折扣策略缓存到 Redis，版本号变更时主动失效。
