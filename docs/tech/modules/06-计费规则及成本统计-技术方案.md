# 计费规则及成本统计模块技术方案

## 1. 架构设计
- 子模块：计费规则、实时计费、账单汇总、对账。
- 组件：
  - `BillingRuleEngine`：规则匹配与费率计算。
  - `CostAggregator`：分钟/小时/日聚合任务。
  - `ReconciliationService`：供应商账单对账。

## 2. 数据设计
- 核心表：`billing_rules`、`gateway_requests`、`billing_statements`。
- 推荐新增：`provider_bills`、`reconciliation_diffs`。

## 3. 接口设计
- `GET/POST /api/billing/rules`
- `GET /api/billing/summary`
- `GET /api/billing/details`
- `POST /api/billing/reconciliation`

## 4. 关键流程
- 实时计费：请求结束 -> token/状态入库 -> 规则引擎算费 -> 更新账单聚合。
- 对账流程：导入供应商账单 -> 按 request_id 对齐 -> 生成差异报告 -> 人工确认冲正。

## 5. 异常与性能
- 成本聚合采用离线任务，避免影响主链路。
- 账单封账后禁止修改，只允许冲正单。
