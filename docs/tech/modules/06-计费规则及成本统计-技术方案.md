# 计费规则模块技术方案

## 1. 架构设计
- 子模块：计费规则、实时计费、供应商账单对账。
- 组件：`BillingRuleEngine` 负责规则匹配与费率计算，`ReconciliationService` 负责供应商账单对账。

## 2. 数据设计
- 核心表：`billing_rules`、`billing_rule_versions`、`billing_rule_version_items`。
- 复杂计费参数优先写入 `config_json`，阶梯计费写入 `ladder_json`，避免为不同供应商报价模型频繁扩表。

## 3. 计费模式
- `TOKEN`：按输入 Token、输出 Token、缓存读取 Token、缓存写入 Token 分别计价，主流文本模型使用。
- `REQUEST`：按请求次数计价，适合轻量接口。
- `IMAGE`：按图片张数、分辨率、质量档位计价。
- `AUDIO`：按音频时长计价，默认按分钟。
- `VIDEO`：按视频生成时长计价，默认按秒。
- `EMBEDDING`：按输入 Token 计价，面向向量模型。
- `TIERED`：按用量阶梯计价，适合供应商阶梯报价。
- `FREE`：免费额度或内部测试模型。
- `CUSTOM`：自定义计费模式，具体计算参数使用 JSON 配置保存，适合临时或非标准供应商计费规则。

## 4. 接口设计
- `GET/POST /api/billing/rules`
- `POST /api/billing/reconciliation`

## 5. 关键流程
- 实时计费：请求结束 -> 提取用量 -> 匹配计费规则 -> 按计费模式计算成本 -> 通过插件事件推送请求记录或成本明细。
- 对账流程：导入供应商账单 -> 按供应商账单维度匹配 -> 生成差异报告 -> 人工确认修正。

## 6. 异常与性能
- 成本聚合采用离线任务，避免影响主链路。
- 账单封账后禁止直接修改，只允许通过冲正记录修正。
