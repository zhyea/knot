# 通知管理模块技术方案

## 1. 架构设计
- 子模块：模板管理、渠道适配、发送编排、重试与升级。
- 组件：`NotifyTemplateEngine`、`ChannelAdapter`、`NotifyScheduler`。

## 2. 数据设计
- 核心表：`notification_templates`、`notification_records`。
- 推荐新增：`notification_policies`、`notification_dedup_keys`。

## 3. 接口设计
- `GET /api/notifications/templates`
- `POST /api/notifications/send`
- `POST /api/notifications/policies`

## 4. 关键流程
- 告警触发 -> 模板渲染 -> 多渠道发送 -> 回执更新 -> 失败重试/升级。

## 5. 异常与性能
- 同一事件窗口内去重，防止告警风暴。
