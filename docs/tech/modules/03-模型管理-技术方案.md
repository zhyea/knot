# 模型管理模块技术方案

## 1. 架构设计
- 子模块：模型档案、模型参数、模型版本、模型测试。
- 组件：
  - `ModelRegistryService`：模型注册与启停。
  - `ModelVersionService`：版本切换、灰度标记、回滚。
  - `ModelTestService`：测试请求与结果对比。

## 2. 数据设计
- 核心表：`models`、`model_versions`。
- 推荐新增：`model_test_records`、`model_tags`、`model_groups`。

## 3. 接口设计
- `GET/POST/PUT /api/models`
- `POST /api/models/{id}/test`
- `POST /api/models/{id}/versions/switch`

## 4. 关键流程
- 版本切换：校验流量窗口 -> 更新活跃版本 -> 刷新路由缓存 -> 记录审计。
- 测试流程：输入 Prompt -> 调用目标模型 -> 存入测试记录 -> 返回耗时/token。

## 5. 异常与性能
- 模型状态机：`DRAFT -> ACTIVE -> DISABLED`，非法跳转拒绝。
- 模型参数使用 JSON Schema 校验，防止配置污染。
