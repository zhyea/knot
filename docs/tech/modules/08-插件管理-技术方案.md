# 插件管理模块技术方案

## 1. 架构设计
- 插件 SPI：`PreProcessPlugin`、`PostProcessPlugin`、`AuditPlugin`。
- 插件容器：负责发现、加载、隔离执行、生命周期管理。

## 2. 数据设计
- 核心表：`plugins`。
- 推荐新增：`plugin_instances`、`plugin_exec_logs`。

## 3. 接口设计
- `GET /api/plugins`
- `POST /api/plugins`
- `PUT /api/plugins/{id}/status`

## 4. 关键流程
- 安装插件 -> 校验签名 -> 注册元数据 -> 启用实例 -> 灰度放量。
- 网关运行事件 -> 插件事件总线 -> 日志增强插件 -> 外部日志平台、消息队列或自定义存储。

## 5. 异常与性能
- 默认 `fail-open`，可对安全类插件启用 `fail-close`。
- 插件耗时、异常率、熔断状态需要可观测。

## 6. 网关日志事件扩展
- `gateway.request.completed`：用于推送请求记录，覆盖请求状态、目标模型、耗时、Token、错误码和计费快照。
- `gateway.routing.switched`：用于推送路由切换日志，覆盖切换前后目标、原因、命中的路由规则和消费者。
- 核心服务只定义事件契约和插件调用时机，不内置请求记录表、切换日志表或管理页面。
