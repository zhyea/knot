# 系统管理模块技术方案

## 1. 架构设计
- 子模块：用户、角色权限、组织、系统配置、节点管理、操作日志。
- 服务划分：
  - `UserService`：账号生命周期与锁定策略。
  - `RbacService`：角色、权限、授权关系。
  - `OpsService`：节点、配置、备份任务。
  - `AuditService`：操作日志写入与检索。

## 2. 数据设计
- 核心表：`users`、`roles`、`permissions`、`user_roles`、`role_permissions`、`departments`、`user_departments`(新增建议)、`operation_logs`、`operation_log_details`。
- 索引：
  - `users(username)` 唯一；
  - `operation_logs(user_id, created_at)`、`operation_logs(trace_id)`。

## 3. 接口设计
- 用户：`GET/POST/PUT /api/system/users`
- 授权：`POST /api/system/users/{id}/roles`
- 日志：`GET /api/system/operation-logs`、`GET /api/system/operation-logs/{id}`

## 4. 关键流程
- 登录后台 -> JWT 鉴权 -> 权限校验 -> 操作执行 -> AOP 生成审计日志 -> 异步写库。
- 增删改操作强制记录 `before_json/after_json`。

## 5. 异常与性能
- 乐观锁：用户状态更新防并发覆盖。
- 日志写入异步化，失败落本地重试队列。
