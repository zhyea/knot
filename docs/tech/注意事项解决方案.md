# 注意事项解决方案（详细设计）

本文件针对 `docs/prd/Init Prd.md` 中“实施注意事项（补充）”逐条给出可落地方案。

## 1. 多租户隔离
- 方案：
  - 所有业务主表增加 `tenant_id`，并纳入联合索引。
  - MyBatis 统一租户拦截器自动拼接 `tenant_id` 条件。
  - 超级管理员可切换租户上下文，普通用户固定租户。
- 验收：
  - 任意查询 SQL 均含租户条件；
  - 跨租户访问返回 `403`。

## 2. 操作记录审计（含前后快照）
- 方案：
  - 写操作必须标注 `@OperationAudit(action=CREATE/UPDATE/DELETE)`。
  - AOP 在事务前后抓取 `before/after`，写入 `operation_log_details`。
  - 对敏感字段进行脱敏策略（字段白名单 + 掩码规则）。
- 验收：
  - 增删改日志均存在快照；
  - 快照字段脱敏符合策略。

## 3. 系统运维数据化
- 方案：
  - 新增运维表：`gateway_nodes`、`system_configs`、`backup_jobs`、`release_versions`。
  - 备份任务由调度器触发，记录成功/失败与恢复点。
- 验收：
  - 节点状态可视化；
  - 备份/恢复链路可演练。

## 4. 认证方式可扩展
- 方案：
  - 认证策略接口：`AuthStrategy`（ApiKey/OAuth2）。
  - `provider_credentials.auth_config_json` 存 OAuth 参数。
  - 定时任务刷新 Token，失败触发告警。
- 验收：
  - 新增认证方式无需改主流程；
  - 到期前自动续期成功率 > 99%。

## 5. 折扣与计费一致性
- 方案：
  - 折扣策略版本化，命中结果写 `provider_discount_snapshots`。
  - 账单以“策略快照”计算，不受后续策略变更影响。
- 验收：
  - 历史账单可复算且一致。

## 6. 频控额度时段化
- 方案：
  - 规则模型：基础阈值 + 时段窗口（高峰/低谷）。
  - `RateLimitService` 运行时按当前时段加载阈值。
  - 额度不足时触发路由降级策略。
- 验收：
  - 高峰低谷限流阈值自动切换；
  - 额度耗尽可自动切备用模型。

## 7. 监控与告警闭环
- 方案：
  - 指标分层：系统指标、业务指标、策略指标（频控/额度/缓存）。
  - 告警状态机：`OPEN -> ACK -> RESOLVED -> CLOSED`。
  - 告警升级：超时未处理自动升级通知。
- 验收：
  - 关键告警 5 分钟内触达；
  - 告警可追踪完整处理链路。

## 8. 对账与结算机制
- 方案：
  - 对账流水：导入账单 -> 自动比对 -> 差异单 -> 人工确认 -> 冲正。
  - 支持导出 Excel/PDF，附差异明细。
- 验收：
  - 对账差异可定位到 request_id；
  - 冲正过程可审计。

## 9. 灰度与回滚前置
- 方案：
  - 发布默认走灰度计划（10/30/50/100）。
  - 异常阈值触发自动回滚，支持人工一键回滚。
- 验收：
  - 任意变更可在分钟级回滚。

## 10. 合规与数据留存
- 方案：
  - 日志分层留存：热数据 90 天，冷数据 180 天+。
  - 导出审批流 + 下载水印 + 操作审计。
- 验收：
  - 满足审计抽查；
  - 导出行为可追责。
