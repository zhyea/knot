# 供应商管理模块产品文档

## 1. 模块目标
- 统一接入多家模型供应商，完成认证、监控、频控、额度、故障切换。

## 2. 功能范围
- 供应商基础信息：名称、类型、地址、状态、联系人。
- 认证管理：API Key/Secret/Token（加密存储）、有效期提醒。
- 配额与限流：供应商维度 QPS/QPM、日/月调用上限、token 上限。
- 异常治理：失败率告警、超时告警、自动切换备用供应商。
- 成本对账：供应商账单导入和核对。
- 折扣管理：按供应商、模型、应用维度配置折扣策略（直减/折扣率/阶梯折扣）。

## 3. 核心业务规则
- 至少保留一个可用供应商，防止全量不可用。
- 凭证过期前 N 天预警（默认 7 天）。
- 达到额度阈值（默认 80%）触发通知。
- 供应商禁用后，新请求不可路由到该供应商。
- 折扣策略需配置生效区间，多个策略冲突时按优先级和范围最小原则命中。
- 折扣变更采用版本化，不允许直接覆盖历史账期规则。

## 4. 关键页面
- 供应商列表页。
- 认证配置页。
- 频控额度配置页。
- 监控与告警页。
- 折扣策略页（规则列表、冲突校验、生效排期）。

## 5. 关键接口（建议）
- `GET /api/providers`
- `POST /api/providers`
- `PUT /api/providers/{id}`
- `PUT /api/providers/{id}/credential`
- `GET /api/providers/{id}/metrics`
- `GET /api/providers/{id}/discount-policies`
- `POST /api/providers/{id}/discount-policies`
- `PUT /api/providers/{id}/discount-policies/{policyId}`

## 6. 指标与验收
- 供应商切换成功率 > 99.9%。
- 认证失败可追踪（带错误码与供应商返回体摘要）。
- 折扣命中计算可复算，账单与策略快照一致。
