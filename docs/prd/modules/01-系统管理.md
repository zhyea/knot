# 系统管理模块产品文档

## 1. 模块目标
- 建立网关平台用户、组织、角色权限和运维配置的统一管理中心。
- 支撑全模块审计追溯、权限闭环、运维稳定。

## 2. 角色与使用方
- 超级管理员：全量配置与授权。
- 运维管理员：系统参数、节点、备份恢复。
- 业务管理员：用户与部门管理、日志查询。

## 3. 功能范围
- 用户管理：账户信息、重置密码、锁定/解锁、状态管理。
- 角色权限：内置角色、自定义角色、资源权限、操作权限。
- 组织机构：部门树、用户归属、部门数据隔离。
- 系统运维：节点管理、全局参数、版本记录、备份恢复。
- 日志管理：访问日志、操作日志、错误日志、系统日志。
- 操作记录方案：统一采集用户操作上下文，支持审计追溯与风控分析。

## 4. 核心业务规则
- 用户名全局唯一，手机号与邮箱可做唯一约束。
- 权限采用 RBAC（用户-角色-权限）模型，支持多角色叠加。
- 操作日志不可物理删除，仅允许归档。
- 节点下线前必须无活跃流量或强制转移流量。
- 操作记录至少保留 180 天，归档后支持按工单申请恢复检索。
- 高风险操作（删除、禁用、密钥重置）必须记录变更前后快照。
- 所有数据增删改操作必须记录变更快照：新增记录 after，删除记录 before，修改记录 before+after。

## 5. 关键页面
- 用户列表页、用户详情抽屉。
- 角色权限配置页（资源树 + 操作点位）。
- 部门组织树页。
- 系统参数页、节点状态页、日志检索页。

## 6. 关键接口（建议）
- `GET /api/system/users`
- `POST /api/system/users`
- `PUT /api/system/users/{id}/status`
- `GET /api/system/roles`
- `POST /api/system/roles`
- `GET /api/system/logs`
- `GET /api/system/operation-logs`
- `GET /api/system/operation-logs/{id}`

## 7. 操作记录实现方案（摘要）
- 采集方式：Controller 拦截 + AOP 注解双通道采集，覆盖同步与异步操作。
- 记录字段：操作人、模块、动作、对象标识、请求参数摘要、响应结果、耗时、IP、UA、TraceId。
- 变更快照：对数据新增/删除/修改强制采集 before/after，作为审计与回溯依据。
- 存储策略：在线库保留近 90 天，历史归档到冷存储（对象存储或归档库）。
- 检索能力：按时间、操作人、模块、对象、结果状态多维过滤。
- 安全要求：敏感字段（密码、密钥、token）必须脱敏或摘要化存储。

## 8. 指标与验收
- 权限命中准确率 100%（无越权）。
- 日志检索 95 分位响应 < 2s（按最近 7 天）。
- 用户管理核心流程（增删改查）可用。
