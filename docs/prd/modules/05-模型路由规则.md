# 模型路由规则模块产品文档

## 1. 模块目标
- 按业务条件自动将请求分发到最优模型/供应商，兼顾可用性、成本与质量。

## 2. 功能范围
- 规则条件：应用、标签、参数、时段、负载、额度剩余。
- 路由策略：固定、权重、负载均衡、故障转移、成本优先。
- 规则管理：新增、编辑、启停、优先级排序、模拟测试。
- 路由监控：匹配率、命中量、切换记录。

## 3. 核心业务规则
- 规则按优先级从高到低匹配，命中即停止。
- 故障转移策略必须配置备用目标。
- 额度不足时自动触发降级规则。
- 规则变更必须记录版本和操作人。

## 4. 关键页面
- 规则列表页。
- 规则编排页（条件 + 动作）。
- 模拟路由测试页。
- 路由监控页。

## 5. 关键接口（建议）
- `GET /api/routing-rules`
- `POST /api/routing-rules`
- `PUT /api/routing-rules/{id}`
- `POST /api/routing-rules/{id}/test`
- `GET /api/routing-rules/switch-logs`

## 6. 指标与验收
- 规则匹配稳定，无循环路由。
- 异常场景可在秒级切换到备用路径。
