# 模型管理模块产品文档

## 1. 模块目标
- 管理模型全生命周期，支撑路由策略、调试测试、版本治理。

## 2. 功能范围
- 模型档案：名称、供应商、类型、版本、状态、参数能力。
- 模型配置：默认温度、上下文长度、超时、重试。
- 模型标签：精度/速度/成本等标签。
- 测试调试：Prompt 测试、输出比对、耗时与 token 统计。
- 版本管理：灰度、回滚、历史版本记录。
- 频控额度：模型维度 QPS/QPM、日/月调用、token 上限。

## 3. 核心业务规则
- 同供应商下模型编码唯一。
- 模型禁用后不再参与路由候选。
- 灰度发布必须配置比例与生效窗口。
- 版本回滚需保留审计记录。

## 4. 关键页面
- 模型列表页。
- 模型配置页。
- 版本管理页。
- 模型调试页。

## 5. 关键接口（建议）
- `GET /api/models`
- `POST /api/models`
- `PUT /api/models/{id}`
- `POST /api/models/{id}/test`
- `POST /api/models/{id}/versions/switch`

## 6. 指标与验收
- 模型测试结果可追溯（入参、出参、耗时、token）。
- 版本切换支持快速回滚（分钟级）。
